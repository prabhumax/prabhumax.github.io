1190  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS ONLY

1201 /*
1202  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
1203  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1204  *
1205  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
1206  *
1207  * Removes the calling task from the ready list and places it both
1208  * on the list of tasks waiting for a particular event, and the
1209  * list of delayed tasks. The task will be removed from both lists
1210  * and replaced on the ready list should either the event occur (and
1211  * there be no higher priority tasks waiting on the same event) or
1212  * the delay period expires.
1213  *
1214  * @param pxEventList The list containing tasks that are blocked waiting
1215  * for the event to occur.
1216  *
1217  * @param xTicksToWait The maximum amount of time that the task should wait
1218  * for the event to occur. This is specified in kernel ticks,the constant
1219  * portTICK_RATE_MS can be used to convert kernel ticks into a real time
1220  * period.
1221  */
1222 void vTaskPlaceOnEventList( const xList * const pxEventList, portTickType xTicksToWait ) PRIVILEGED_FUNCTION;
1223 
1224 /*
1225  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
1226  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1227  *
1228  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
1229  *
1230  * This function performs nearly the same function as vTaskPlaceOnEventList().
1231  * The difference being that this function does not permit tasks to block
1232  * indefinitely, whereas vTaskPlaceOnEventList() does.
1233  *
1234  * @return pdTRUE if the task being removed has a higher priority than the task
